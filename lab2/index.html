Respuestas a preguntas

¿Cuál es la diferencia entre Internet y World Wide Web?

El internet es una inmensa red de computadoras alrededor del mundo conectadas entre sí y la World Wide Web es una enorme colección de páginas que es creada a base de esa red de computadoras. (BBC News Mundo, 2019)

¿Cuáles son las partes de un URL?

Los componentes de una URL se delimitan como: scheme://hostname:port/path?query El hostname es la dirección IP seguida de un puerto y al último un componente de vía de acceso. (IBM, s.f.)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Son usados para permitir a los clientes API a realizar acciones CRUD. GET: Usado para recuperar datos en un servidor. POST: Usado para crear nuevos recursos. PUT: Usado para reemplazar un recurso existente con una versión actualizada. PATCH: Usado para actualizar parcialmente un recurso existente. DELETE: Usado para remover datos de la base de datos. (Postman, 2023)

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST, debido a que se están introduciendo datos como lo es el nombre de usuario y contraseña.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET, ya que se usa para recuperar datos del servidor

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, no ha ocurrido ningún error, al contrario la petición por parte del usuario se ha realizado con éxito. El código 200 da como resultado OK, los códigos de 200-299 son respuestas satisfactorias.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No, un error 404 significa que el servidor no pudo encontrar el contenido solicitado “Not Found”. Los errores de 400-499 son errores de parte del cliente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, el error 500 significa que el servidor ha tenido error al realizar la solicitud, refiriendo que el error viene por parte del desarrollador. (mdn, s.f.)

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando un elemento es depreciado significa que su uso ya no es recomendable debido a que en futuras actualizaciones será eliminada por completo. (ManzDev, s.f.)

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 ofrece soporte nativo para multimedia, elementos semánticos, APIs avanzadas y sintaxis simplificada.

¿Qué componentes de estructura y estilo tiene una tabla?

Tiene los siguientes componentes: table, caption, thead, colgroup, col, th, tbody, tr, td y tfoot

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles son: input, textarea, select>/option para listas, y button para acciones.

Respuestas a preguntas CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Mi recomendación del uso de !important es que se use con disciplina porque al usarlo se ignora la jerarquía de cascada normal, si no se usa con cuidado podría resultar en más conflictos que beneficios.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Se debe escoger con cuidado porque puede afectar la legibilidad del contenido y el rendimiento de la página. Además, si la imagen es muy pesada, hará que la página cargue más lento.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Todo depende del uso que se le quiera dar a la página, si por ejemplo, estamos haciendo una página responsiva para que sea usada en PC, móvil y tableta, el uso de % sería la mejor opción porque toma en cuenta la proporción de la pantalla. Las unidades de px y pt tienen una función similar pero igual funcionan para dar estilo, pero no se recomienda si se están haciendo una página responsiva.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Porque tiene menos datos que descargar al estar cargando la página.
