¿Por qué es buena práctica usar JavaScript para validar antes de enviar al servidor?

Porque mejora la experiencia del usuario y evita enviar datos incorrectos innecesariamente al servidor.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Desactivando JavaScript, modificando el código desde el navegador o enviando peticiones manuales con herramientas como Postman.

Si se pueden saltar, ¿por qué sigue siendo buena práctica?

Porque la validación en el cliente mejora la experiencia y eficiencia, pero la seguridad real siempre debe hacerse también en el servidor. Son complementarias, no excluyentes.
